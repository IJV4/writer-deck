# Group 1 тАФ Correctness bugs (`FIX-*`)

## [x] FIX-1 ЁЯФ┤ тАФ `.md` documents load empty and get silently forked to `.txt`

- **Files:** `writerdeck/utils/file_manager.py`.
- **Anchor:** `def _doc_path(self, name: str) -> Path:` and `def rename(self, old_name, new_name)`.
- **Symptom:** `list_documents`/`list_entries`/`most_recent_document` surfaced `*.md` files, but
  `_doc_path` always returned `name + ".txt"` and `load` only probed `.txt`/`.autosave`. Opening a
  document that existed only as `name.md` loaded an **empty** buffer; the next save wrote `name.txt`,
  orphaning the original `.md` content тАФ a data-loss class reachable through the normal file picker.
- **Fix:** added `_DOC_SUFFIXES = (".txt", ".md")`. `_doc_path` now probes the known suffixes and
  returns the first that exists (else `.txt` for a brand-new doc), so a `.md`-only doc loads and
  **saves back to its `.md`**. `rename` preserves the source extension via
  `self._doc_path(new_name).with_suffix(old_path.suffix)`.
- **Tests:** `tests/test_file_manager.py::TestFileManagerMarkdown` (6 tests: load `.md`-only, save
  preserves `.md`, new doc тЖТ `.txt`, `.txt` preferred when both exist, `.last_open` finds `.md`,
  rename preserves `.md`).
- **Verify:** `pytest tests/test_file_manager.py` тЖТ **61 passed** (ran on this machine; PIL/yaml-free).

## [x] FIX-2 ЁЯФ┤ тАФ `deploy.sh` prune deletes the wrong release

- **Files:** `deploy.sh` (the prune loop in the deploy remote heredoc).
- **Anchor:** was `ls -1 "$RELEASES" | sort | while IFS= read -r name; do`.
- **Symptom:** once a one-time `setup-<ts>` bootstrap release coexisted with more than `KEEP=5`
  releases, the prune used a plain lexical `sort`. Because `'s' > '2'`, `setup-<ts>` sorted **newest**
  and was kept forever, while a genuinely newer `2026тАж` release was pruned in its place тАФ the exact
  trap the rollback block already documented and avoided.
- **Fix:** the prune now sorts oldest-first by the trailing 15-char `YYYYMMDD-HHMMSS` timestamp,
  identical to the rollback block:
  `ls -1 "$RELEASES" | while IFS= read -r d; do printf '%s %s\n' "${d: -15}" "$d"; done | sort | cut -d' ' -f2-`.
- **Verify:** `REPRO-VERIFIED` via a local simulation with a `setup-<ts>` + six timestamped releases
  (`KEEP=5`): old logic pruned `20260711-120000` (a newer release) and kept `setup-*`; new logic
  correctly prunes the two genuinely-oldest. **No shell test harness exists** тАФ see ENV-PENDING for a
  suggested follow-up (`shellcheck` + extracting the sort into a testable helper).

## [x] FIX-3 ЁЯЯа тАФ Renaming the active document doesn't repoint `.last_open`

- **Files:** `writerdeck/core/app.py`.
- **Anchor:** `_handle_overlay_result`, the `elif "renamed" in result:` branch.
- **Symptom:** the branch updated `self._doc.name` but never called `save_last_open(new)` тАФ unlike the
  `open_doc`/`save_as`/`deleted` branches. `.last_open` kept the old name, so after a reboot the
  renamed doc wasn't reopened (silent fall back to `most_recent_document`).
- **Fix:** call `self._file_mgr.save_last_open(new)` inside the `if self._doc.name == old:` guard
  (only when the renamed doc is the active one; renaming an inactive doc leaves `.last_open` alone).
- **Tests:** `tests/test_app.py::TestOverlayDispatch::test_handle_overlay_result_rename_repoints_last_open`
  and `..._rename_inactive_leaves_last_open`.
- **Verify:** `ENV-PENDING` тАФ `pytest tests/test_app.py` (imports `PIL` via the render path).

## [x] FIX-4 ЁЯЯа тАФ `id(font)`-keyed caches are never cleared on font change

- **Files:** `writerdeck/display/glyph_cache.py`, `writerdeck/utils/text_wrapper.py`,
  `writerdeck/core/app.py`.
- **Anchor:** `_glyph_cache` (glyph_cache), `_char_width_cache` + `def clear_wrap_cache()`
  (text_wrapper), the `elif "font" in result:` branch (app).
- **Symptom:** both caches key on `id(font)` and were never cleared тАФ the font-change handler only
  cleared `_wrap_cache`. Because `fonts.get_font` is `lru_cache(maxsize=16)`, an evicted + GC'd font's
  address can be reused by a *different* font, at which point a stale entry renders the wrong
  glyph/width. (`CLAUDE.md`'s "cleared on font change" was true only for `_wrap_cache`.)
- **Fix:** added `clear_glyph_cache()` to `glyph_cache.py`; `clear_wrap_cache()` now also clears
  `_char_width_cache`; the app font-change handler calls both `clear_wrap_cache()` and
  `clear_glyph_cache()`.
- **Tests:** `tests/test_glyph_cache.py` (new тАФ clear empties cache, cached ink is drawn, width
  stable) and `tests/test_text_wrapper.py::test_clear_wrap_cache_also_clears_char_width_cache`.
- **Verify:** `ENV-PENDING` тАФ both import `PIL`.

## [x] FIX-5 ЁЯЯа тАФ `get_font` can raise `OSError` into the render loop

- **Files:** `writerdeck/display/fonts.py`.
- **Anchor:** `@lru_cache тАж def get_font(...)`, last-resort `ImageFont.truetype("DejaVuSansMono.ttf", size)`.
- **Symptom:** the last-resort load raised `OSError` on a minimal Pi image with no DejaVu. The render
  path only catches `DisplayError`, so this would crash the main loop instead of degrading headless.
- **Fix:** layered fallback that never raises тАФ found path тЖТ bundled `assets/fonts/Hack-Regular.ttf`
  тЖТ `DejaVuSansMono.ttf` тЖТ `ImageFont.load_default(size)` (with a `TypeError` guard for Pillow 10.0).
  Return annotation widened to `FreeTypeFont | ImageFont`; added a `logging` warning on fallback.
- **Tests:** `tests/test_fonts.py` (new тАФ usable font, unknown family falls back, and a monkeypatched
  "all truetype fails" case still returns a font).
- **Verify:** `ENV-PENDING` тАФ imports `PIL`.

## [x] FIX-6 ЁЯЯа тАФ Two validated config knobs do nothing

- **Files:** `writerdeck/core/config.py`, `config_default.yaml`, `USER_GUIDE.md`, `NEXT-STEPS.md`.
- **Anchor:** `_SCHEMA` entries + accessors `render_interval_ms` / `display_sleep_minutes`.
- **Symptom:** `render_interval_ms` was validated + documented but never read (the loop uses
  `self._hw.render_interval_ms` from the hardware profile). `display_sleep_minutes` had **zero**
  consumers (the real trigger is `display_idle_sleep_seconds`), yet the docs told users to tune it for
  battery life.
- **Decision:** per the maintainer, **remove the redundancy** rather than wire it (a deliberate power
  mode can be built later if wanted). Deleted both keys from `_SCHEMA`, both accessor properties, and
  both lines from `config_default.yaml`.
- **Docs:** rewrote `USER_GUIDE.md`'s "All Options" block тАФ removed the two dead knobs and added the
  real display knobs that were missing (`full_refresh_max_seconds`, `display_idle_sleep_seconds`,
  `display_screensaver_seconds`, `enable_perf_metrics`); pointed the sleep guidance at
  `display_idle_sleep_seconds`. Fixed the `NEXT-STEPS.md` battery-tuning line.
- **Tests:** removed `test_render_interval_ms`/`test_display_sleep_minutes`, dropped the keys from the
  fixtures, kept `test_multiple_errors` meaningful (swapped to `autosave_interval_seconds`).
- **Verify:** `ENV-PENDING` тАФ `pytest tests/test_config.py` (imports `PyYAML`). Note the
  `HardwareProfile.render_interval_ms` field and `app.py`'s `self._hw.render_interval_ms` are the
  correct hardware-side values and were intentionally **kept**.

# Group 2 тАФ Tech debt (`TD-*`)

## [x] TD-1 ЁЯЯа тАФ Markdown parser silently drops a lone `*`

- **Files:** `writerdeck/utils/markdown.py` (`_parse_inline`).
- **Fix:** regex changed to `r"\*\*(.+?)\*\*|\*(.+?)\*|([^*]+|\*)"` тАФ the final alternative now also
  matches a solitary `*`, so an unpaired asterisk (`a * b`, `2 * 3 = 6`) survives verbatim instead of
  being dropped to `a  b`. `**bold**`/`*italic*` pairs still parse as before.
- **Tests:** `tests/test_markdown.py::test_lone_asterisk_preserved`, `test_arithmetic_asterisk_preserved`.
- **Verify:** `pytest tests/test_markdown.py` тЖТ ran on this machine, green (PIL/yaml-free).

## [x] TD-2 ЁЯЯа тАФ Byte-identical paged-input duplicated across modes

- **Files:** `writerdeck/modes/base_mode.py`, `distraction_free.py`, `dashboard.py`, `typewriter.py`.
- **Fix:** hoisted the shared PAGE_PREV/PAGE_NEXT + `_handle_visual_updown` + `_apply_common_input`
  block into `BaseMode._handle_paged_input`. `DistractionFreeMode` and `DashboardMode` now delegate to
  it. `TypewriterMode` is **intentionally left** with its own (scroll-based) handler тАФ a comment now
  documents that Ctrl+Up/Down are no-ops there by design (behavior unchanged).
- **Verify:** `ENV-PENDING` тАФ `pytest tests/test_pagination.py tests/test_mode_scroll_selection.py`
  (import `PIL` via `text_wrapper`). Behavior-preserving refactor; confirmed by code-trace + review.

## [x] TD-3 ЁЯЯа тАФ `stdin` dev backend missing keys

- **Files:** `writerdeck/input/stdin_reader.py` (`_ESCAPE_SEQUENCES`).
- **Fix:** added `\x1b[1;5A`/`\x1b[1;5B` тЖТ `PAGE_PREV`/`PAGE_NEXT` (Ctrl+Up/Down) and `\x1b[Z` тЖТ
  `SWITCH_MODE_PREV` (Shift+Tab), giving desktop dev parity with the evdev page-nav and reverse
  mode-cycle. (`pygame` already routes through `KeyMapper`, so it needed nothing.) `DELETE_WORD_BACK`
  and `FONT_MENU` remain unmapped in `stdin` тАФ they're genuinely ambiguous/undistinguishable in a raw
  terminal; noted, not forced.
- **Tests:** `tests/test_stdin_reader.py` тАФ dict assertions + parse tests for the new sequences.
- **Verify:** `pytest tests/test_stdin_reader.py` тЖТ ran on this machine, green (PIL/yaml-free).

## [x] TD-4 тЪк тАФ `os.system(...)` тЖТ `subprocess.run(...)`

- **Files:** `writerdeck/core/app.py` (`_enter_system_suspend`), `writerdeck/utils/power.py`
  (critical-battery poweroff).
- **Fix:** both now use `subprocess.run([...], check=False, capture_output=True, text=True)` with a
  logged warning/error on a nonzero return code (previously the exit code was silently discarded).
  Dropped the now-unused `import os` from `power.py`; kept it in `app.py` (still used for
  `os.environ`/`os.path.exists`).
- **Test-mock landmine (important):** `test_power.py` **and** `test_app.py` both patched `os.system`
  and asserted the command string. Left unfixed those tests would break тАФ and a mispatched
  `subprocess.run` could let a **real** `poweroff`/`suspend` through in CI. Both were repointed to
  `writerdeck.тАжsubprocess.run` (with `returncode = 0`) and switched to assert the argument list.
- **Verify:** `pytest tests/test_power.py` тЖТ ran on this machine, green (274 in that batch).
  `app.py` half is `ENV-PENDING` (`test_app.py` imports `PIL`).

## [x] TD-5 тЪк тАФ `load_config` ignores an explicit `project_root` after first call

  explicit `project_root` always (re)loads from that tree and refreshes the singleton, so a

- **Tests:** `tests/test_config.py::TestLoadConfig::test_explicit_root_reloads_over_singleton`.
- **Verify:** `ENV-PENDING` тАФ imports `PyYAML`.

# Group 3 тАФ Deliberately NOT changed (`SKIP-*`)

These touch hardware-critical, "Do NOT touch (by-design)" code (see
[`IMPROVEMENTS-PLAN.md` ┬з Do NOT touch](IMPROVEMENTS-PLAN.md)); a regression here is invisible to
desktop tests. Decision (2026-07-13): **leave all three.**

- **SKIP-1 тАФ driver post-wake full-refresh self-defense.** `EPaperDriver.wake()` relies on `App`
  forcing a full refresh (`request_full()`), rather than self-defending. Already enforced at the App
  level; the safe driver-side fix would touch the protected `_last_buf` preservation. **Left.**
- **SKIP-2 тАФ glyph-cache `mask_img.im = mask_core`.** A private-PIL-attribute hack, but there is no
  clean public API to build an `Image` from a font mask core, and it's load-bearing + documented.
  Replacing it risks a panel-only rendering regression. **Left.**
- **SKIP-3 тАФ desktop partial-diff coverage.** The pure diff functions (`compute_dirty_bands`,
  `compute_x_window`) are already unit-tested; making `NullDriver` track `_last_buf` wouldn't exercise
  the real `EPaperDriver` path. **Left.**
